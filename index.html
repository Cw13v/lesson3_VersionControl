Highlight #1 Version Control

A version control is the main repository that stores all project files. This is done so team members can check files and make changes to their own copy. Once they have finished making their changes they place it back into the version control for the rest of the team members to see and work on. This is done to help stop other team members accidental deleting your files while they work on their files. It gives users a revision history so if changes are made that need to be undone you can go back to see where the change happened.

Highlight #2 Remote and Local Repository

A remote and local repository work together. A remote is the common repository that all developers have access to. The local is each developers own repository that resides in their computer. A developer pulls their files off the remote repository to work on. They edit the file and make the necessary changes to the files. After they have completed editing they add the files back to the remote repository. Once they have finished uploading the files they are now on the remote repository for other developers to see. Other developers can pull the newly edited files off the remote repository to edit on their own local repository.

Highlight #3 GIT

A GIT is one of the main version control systems. It is a free open source system that can handle any size project. It is used by a range of companies including Google, Facebook, Twitter, and Netflix. It is a distributed version control system. This means everything is stored locally on a developers own computer. This means that if the developer is offline they can still work on their project at any point. GIT runs much faster than a remote repository. Some of the comands GIT has are GIT log, GIT add, and GIT remote.
